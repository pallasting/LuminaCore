# 可视化的流程图

可视化的流程图，是向外界展示你设计理念最有力的工具。

为了方便你直接用于文档或演示，我为你准备了两种形式的图表：

1. **Mermaid 代码版**（专业的流程图代码，你可以直接复制到支持Markdown的编辑器或Notion中生成矢量图）。
2. **ASCII 字符版**（直观展示，所见即所得）。

---

### 版本一：Mermaid 流程图代码 (推荐用于正式文档)

你可以将以下代码复制到任何支持 Mermaid 的编辑器（如 Notion, Obsidian, GitHub 或在线编辑器 mermaid.live）中，它将生成一个专业的架构图。

```mermaid
graph LR
    %% 定义样式
    classDef elec fill:#f9f,stroke:#333,stroke-width:2px,color:black;
    classDef opt_source fill:#ffcccb,stroke:#f00,stroke-width:2px,color:black;
    classDef waveguide fill:#e0e0e0,stroke:#666,stroke-width:2px,stroke-dasharray: 5 5;
    classDef passive fill:#d4f1f4,stroke:#00bcd4,stroke-width:2px,color:black;
    classDef compute fill:#c8e6c9,stroke:#4caf50,stroke-width:2px,color:black;

    %% 1. 控制层 (Electronics)
    subgraph Control_Layer [Step 1: 电子控制层 (System 1)]
        V_Control[CPU/MCU 控制器]:::elec
        V_R[电压 V1 (权重/开关)]:::elec
        V_G[电压 V2 (权重/开关)]:::elec
        V_B[电压 V3 (权重/开关)]:::elec
    end

    %% 2. 光源层 (Source)
    subgraph Source_Layer [Step 2: 稀土纳米晶阵列 (Nature 2025)]
        Pixel_R((红光像素 R)):::opt_source
        Pixel_G((绿光像素 G)):::opt_source
        Pixel_B((蓝光像素 B)):::opt_source
    end

    %% 3. 传输与路由 (Router)
    subgraph Transport_Layer [Step 3: 波分复用传输 & 路由]
        Main_WG(主光波导\n多色混合光束):::waveguide
        Grating{无源光栅\n(AWG 路由器)}:::passive
    end

    %% 4. 计算层 (Computing)
    subgraph Compute_Layer [Step 4: 并行计算光路]
        Path_A[光路 A: 矩阵加法]:::compute
        Path_B[光路 B: 矩阵减法]:::compute
        Path_C[光路 C: 矩阵乘法]:::compute
    end

    %% 5. 输出层 (Output)
    Result(探测器 / 下一级计算):::elec

    %% 连接关系
    V_Control --> V_R & V_G & V_B
    V_R -->|驱动| Pixel_R
    V_G -->|驱动| Pixel_G
    V_B -->|驱动| Pixel_B

    Pixel_R & Pixel_G & Pixel_B == 耦合 ==> Main_WG
    Main_WG == RGB混合光束 ==> Grating

    Grating -- 红光 (λ1) --> Path_A
    Grating -- 绿光 (λ2) --> Path_B
    Grating -- 蓝光 (λ3) --> Path_C

    Path_A & Path_B & Path_C -.-> Result

    %% 注释
    style Control_Layer fill:#fff,stroke:#333,stroke-dasharray: 5 5
```

### 版本二：ASCII 字符可视化版 (直观理解)

这个图展示了当你想**“同时执行加法和减法”**（即混合光模式）时，光子在你的芯片内部是如何流动的。

    | V2: 0.8V  |=======>| (G) 绿像素  | =========>|            |            +----------------+

                                               /   |   (WDM)    |

    +-----------+        +-------------+           |            |            | Matrix Sub     |
       (OFF)                (不发光)                +------------+             +----------------+
                                                         |

                                                         v
                                                   +----------------+
                                                   | [通道 C]       |
                                                   | 矩阵乘法单元   |
                                                   | (本轮闲置)     |
                                                   +----------------+

--------------------------------------------------------------------------------
结果解析：
1. 输入：你输入了 V1(高) 和 V2(中)。
2. 现象：主波导中传播的是 "橙黄色" 的光 (红+绿混合)。
3. 路由：光栅不需要电，它利用物理定律，自动把红光踢给A，把绿光踢给B。

### 图表设计说明（用于你的文档描述）

在你的设计文档中，你可以配合上述图表添加以下**技术注解**，这将大大提升文档的专业度：

1. **输入层的“双重含义”：**
    * 电压的**“有/无”**（On/Off）代表了**逻辑控制**（选择哪个计算单元）。
    * 电压的**“幅值”**（Amplitude）代表了**模拟权重**（计算时的参数大小）。

2. **传输层的“时空压缩”：**
    * 图中的 **"主波导 (WDM)"** 是核心瓶颈的突破口。它展示了如何在一个物理通道内，通过颜色维度（$\lambda_1, \lambda_2, \lambda_3$）堆叠多个数据流。

3. **路由层的“零能耗优势”：**
    * 重点标注光栅（Grating）部分为 **"Passive Component"（无源器件）**。这是与电子芯片（需要晶体管开关）最大的区别，也是你低功耗架构的来源。

4. **计算层的“可重构性”：**
    * 虽然光路A、B、C是固定的，但通过改变源头的电压组合，整个系统的**功能**是可重构的（Reconfigurable）。

这一套图表配合文字，已经能够非常清晰地向资深工程师或投资人解释清楚 **LuminaCore** 到底是如何工作的了。
